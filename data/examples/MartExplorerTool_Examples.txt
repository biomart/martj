The following are a few examples for using the MartExplorerTool commandline interface to Ensembl Mart.
These require that your CLASSPATH be set correctly (Note, You can substitute >/pathto/mart-explorer/bin/martcl.sh
for >java org.ensembl.mart.explorer.MartExplorerTool, and the shell script will set your CLASSPATH for each query).

# connect to kaka.sanger.ac.uk and get a list of gene_stable_id's from ensembl_mart_9_1 for homo_sapiens gene (core)
>java org.ensembl.mart.explorer.MartExplorerTool -H kaka.sanger.ac.uk -u anonymous -d ensembl_mart_9_1 -s homo_sapiens -c gene -a gene_stable_id

# use file $HOME/.martexplorer to configure connection to your personal mysql database and mart database name.
>java org.ensembl.mart.explorer.MartExplorerTool -s homo_sapiens -c gene -a gene_id

# use a different connection configuration file than $HOME/.martexplorer
>java org.ensembl.mart.explorer.MartExplorerTool -g /pathto/myconnectionconf_file -s homo_sapiens -c gene -a gene_id

# connect get a list of gene_id's from a list of gene_stable_id's in a file (note, you must specify the file as a URL, eg with a file: protocal)
>java org.ensembl.mart.explorer.MartExplorerTool -s homo_sapiens -c gene -I file:pathto/filewithids -t gene_stable_id -a gene_id

# pipe a list of gene_stable_ids (eg, from another program) into a MartExplorerTool query as a gene_stable_id filter to get a list of gene_ids
>cat data/gene_stable_ids.test | java org.ensembl.mart.explorer.MartExplorerTool -s homo_sapiens -c gene -S -t gene_stable_id -a gene_id

# pipe the gene_id output of the previous MartExplorerTool query into another MartExplorerTool query as a gene_id filter, and get the gene_stable_ids back.
>cat data/gene_stable_ids.test | java org.ensembl.mart.explorer.MartExplorerTool -s homo_sapiens -c gene -S -t gene_stable_id -a gene_id | java org.ensembl.mart.explorer.MartExplorerTool -s homo_sapiens -c gene -S -t gene_id -a gene_stable_id
